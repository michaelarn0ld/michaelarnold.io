Basics of Stdin, Stdout, Stderr.

Standard input (stdin), standard output (stdout), and standard error (stderr) are three streams out data that are create when you launch a program. They describe what input a program recieves, what data it produces, and any errors that occur during execution.

When you enter a command, your keyboard input is the presumed input unless a file is specified. Similarly, the screen serves as standard output; commands take input and send the results, if there are any, to the ouput. Finally, standard error is where program errors are delivered; by default, this is the screen.

Pipes and Stdin

The power of UNIX comes from the ability to chain (pipe) the stdout of one program to the stdin of another. Consider the following example:

  grep '#' README.md | sed 's/#*//g' | nl | more

Let's dissect the steps of this command. First, we send the file README.md to the stdin of grep and select all lines containing "#". Next, we send the output of that program to stdin of sed and substitute any "#" for nothing (in other words we remove the character "#" entirely). Then, we send the output of this to the stdin of nl, adding line numbers to each line of output. Finally, we send the output to more which sends its output to the screen. Consider the following README.md:

    #foo
    ##bar
    //this line omitted
    ###foo##bar

Which when fed to our above command produces the output:

    1. foo
    2. bar
    3. foobar

Redirecting Stdout and Stderr

  find / > find.out 2>found.err

This command recursively finds all files rooted from /, redirects stdout to find.out, and redirects stderr to find.err. It is exactly equivalent to: 

  find / 1> find.out 2>found.err

This shows us the concept of file descriptors; standard output has the file descriptor of 1, while standard error has the file descriptor of 2. If you want to write both the output and errors to the same location, then use: 

  find / > find.both 2>&1

In english, this command instructs us to first redirect output to find.both and then redirect the errors to the output.
